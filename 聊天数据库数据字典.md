# 聊天 SQLite 数据库数据字典

本文说明 mesh-proxy 聊天模块（`internal/chat`）使用的 **SQLite** 数据库结构。默认文件路径为 `{DataDir}/chat.db`（见 `internal/app/app.go` 与 `chat.NewService`）。

**实现参考**：`internal/chat/store.go`（迁移与单聊表）、`internal/chat/group_store.go`（群聊表）、`internal/chat/types.go` / `internal/chat/group_types.go`（常量与枚举值）。

**时间字段**：多数 `*_at` 为 **RFC3339Nano** 字符串（UTC），由 `formatDBTime` / `time.RFC3339Nano` 写入。

**索引**：`messages` 上有 `idx_messages_conversation_created_at`、`idx_messages_conversation_counter`；群聊表索引见各表「备注」。

---

## 1. 单聊与通讯录相关

### 1.1 `profile`

本机用户的聊天设置与密钥数据（每库通常一行，主键为本机 `peer_id`）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `peer_id` | TEXT PK | 本机节点 ID |
| `nickname` | TEXT | 昵称 |
| `bio` | TEXT | 简介（默认 `''`） |
| `avatar` | TEXT | 头像识别（名称或路径等，默认 `''`） |
| `chat_kex_priv` | TEXT | 聊天密钥交换私钥（存储格式依实现） |
| `chat_kex_pub` | TEXT | 聊天密钥交换公钥 |
| `created_at` | TEXT | 建立时间 |
| `updated_at` | TEXT | 更新时间 |

---

### 1.2 `peers`

已知对端联系人缓存（非本机 profile）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `peer_id` | TEXT PK | 对端节点 ID |
| `nickname` | TEXT | 昵称 |
| `bio` | TEXT | 简介 |
| `avatar` | TEXT | 头像识别 |
| `blocked` | INTEGER | 是否封锁：`0` 否、`1` 是 |
| `last_seen_at` | TEXT | 最后上线／可见时间（可为空字符串） |
| `updated_at` | TEXT | 更新时间 |

---

### 1.3 `requests`

好友／会话请求（含发起与接收），对应 `Request` 结构。

| 字段 | 类型 | 说明 |
|------|------|------|
| `request_id` | TEXT PK | 请求唯一 ID |
| `from_peer_id` | TEXT | 发起方 peer |
| `to_peer_id` | TEXT | 接收方 peer |
| `state` | TEXT | 状态，见 **§4.1 好友请求状态** |
| `intro_text` | TEXT | 附言／介绍 |
| `nickname` | TEXT | 请求时展示的昵称 |
| `bio` | TEXT | 请求时展示的简介 |
| `avatar` | TEXT | 请求时展示的头像识别 |
| `retention_minutes` | INTEGER | 消息保留分钟数（`0` 表示默认或未设置） |
| `remote_chat_kex_pub` | TEXT | 对端聊天公钥 |
| `conversation_id` | TEXT | 接受后关联的会话 ID（默认 `''`） |
| `last_transport_mode` | TEXT | 最后传输模式，见 **§4.3** |
| `created_at` | TEXT | 建立时间 |
| `updated_at` | TEXT | 更新时间 |

---

### 1.4 `conversations`

单聊会话，每位 `peer_id` 至多一笔（`peer_id` UNIQUE）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversation_id` | TEXT PK | 会话 ID |
| `peer_id` | TEXT UNIQUE | 对端 peer |
| `state` | TEXT | 见 **§4.2 会话状态** |
| `last_message_at` | TEXT | 最后一则消息时间（无消息可为空字符串） |
| `last_transport_mode` | TEXT | 见 **§4.3** |
| `unread_count` | INTEGER | 未读数 |
| `retention_minutes` | INTEGER | 本地消息保留分钟数 |
| `retention_sync_state` | TEXT | 保留策略与对端同步状态：`synced`、`pending` 等 |
| `retention_synced_at` | TEXT | 对端确认同步的时间（未确认可为空字符串） |
| `created_at` | TEXT | 建立时间 |
| `updated_at` | TEXT | 更新时间 |

---

### 1.5 `messages`

单聊消息本体；`plaintext` 为解密后缓存，`ciphertext_blob` 为密文。

| 字段 | 类型 | 说明 |
|------|------|------|
| `msg_id` | TEXT PK | 消息 ID |
| `conversation_id` | TEXT FK 语义 | 所属会话 |
| `sender_peer_id` | TEXT | 发送方 |
| `receiver_peer_id` | TEXT | 接收方 |
| `direction` | TEXT | `inbound` / `outbound` |
| `msg_type` | TEXT | 协议消息类型，见 **§4.4**（`types.go` 常量） |
| `plaintext` | TEXT | 解密后内容（或占位） |
| `file_name` | TEXT | 文件名（非文件消息为 `''`） |
| `mime_type` | TEXT | MIME |
| `file_size` | INTEGER | 文件大小（字节） |
| `ciphertext_blob` | BLOB | 密文 |
| `transport_mode` | TEXT | 见 **§4.3** |
| `state` | TEXT | 传递状态，见 **§4.5 单聊消息状态** |
| `counter` | INTEGER | 会话内序号（加密／同步用 `uint64`） |
| `created_at` | TEXT | 建立时间 |
| `delivered_at` | TEXT | 对端送达确认时间（未确认为 `''`） |

---

### 1.6 `session_states`

单聊端到端会话的对称密钥与计数器（`send_key` / `recv_key` 以 **标准 Base64** 字符串存储）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversation_id` | TEXT PK | 会话 ID |
| `peer_id` | TEXT | 对端 peer |
| `send_key` | TEXT | 发送方向密钥（Base64） |
| `recv_key` | TEXT | 接收方向密钥（Base64） |
| `send_counter` | INTEGER | 发送计数 |
| `recv_counter` | INTEGER | 接收计数 |

---

## 2. 异步工作队列（单聊）

### 2.1 `outbox_jobs`

待重试发送的单聊消息；`job_id` 通常等于 `msg_id`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `job_id` | TEXT PK | 与消息 ID 对应 |
| `peer_id` | TEXT | 目标 peer |
| `msg_id` | TEXT | 消息 ID |
| `status` | TEXT | 与消息状态语义一致，常见：`queued_for_retry`、`sent_to_transport`（见 **§4.5**） |
| `retry_count` | INTEGER | 重试次数 |
| `next_retry_at` | TEXT | 下次重试时间 |
| `last_transport_attempt` | TEXT | 上次传输尝试时间 |

---

### 2.2 `message_revoke_jobs`

撤回消息的传送重试队列。

| 字段 | 类型 | 说明 |
|------|------|------|
| `job_id` | TEXT PK | 通常等于 `msg_id` |
| `conversation_id` | TEXT | 会话 ID |
| `peer_id` | TEXT | 对端 peer |
| `msg_id` | TEXT | 被撤回的消息 ID |
| `status` | TEXT | 如 `queued_for_retry`、`sent_to_transport` |
| `retry_count` | INTEGER | 重试次数 |
| `next_retry_at` | TEXT | 下次重试时间 |
| `last_transport_attempt` | TEXT | 上次传输尝试时间 |
| `created_at` | TEXT | 入列时间 |

---

### 2.3 `friend_request_jobs`

好友请求发送／重试状态（`job_id` 对应 `requests.request_id`）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `job_id` | TEXT PK | 请求 ID |
| `peer_id` | TEXT | 对端 peer |
| `status` | TEXT | 重试枚举时使用 `queued_for_retry`（见 `ListFriendRequestJobsForRetry`） |
| `retry_count` | INTEGER | 重试次数 |
| `next_retry_at` | TEXT | 下次重试时间 |
| `last_transport_attempt` | TEXT | 上次传输尝试时间 |
| `created_at` | TEXT | 建立时间 |

---

## 3. 群聊相关

### 3.1 `groups`

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | TEXT PK | 群组 ID |
| `title` | TEXT | 群名称 |
| `avatar` | TEXT | 群头像识别 |
| `controller_peer_id` | TEXT | 群主 peer |
| `current_epoch` | INTEGER | 目前密钥世代 |
| `retention_minutes` | INTEGER | 消息保留分钟数 |
| `state` | TEXT | **§4.6 群状态**：`active`、`archived` |
| `last_event_seq` | INTEGER | 最后事件序号 |
| `last_message_at` | TEXT | 最后消息时间 |
| `created_at` | TEXT | 建立时间 |
| `updated_at` | TEXT | 更新时间 |

索引：`idx_groups_updated_at`。

---

### 3.2 `group_members`

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | TEXT PK 复合 | 群组 ID |
| `peer_id` | TEXT PK 复合 | 成员 peer |
| `role` | TEXT | **§4.7 群角色**：`controller`、`admin`、`member` |
| `state` | TEXT | **§4.8 成员状态**：`invited`、`active`、`left`、`removed` |
| `invited_by` | TEXT | 邀请人 peer |
| `joined_epoch` | INTEGER | 加入时世代 |
| `left_epoch` | INTEGER | 离开时世代 |
| `updated_at` | TEXT | 更新时间 |

索引：`idx_group_members_state`。

---

### 3.3 `group_events`

群控制事件账本，`(group_id, event_seq)` 唯一。

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | TEXT PK | 事件 ID |
| `group_id` | TEXT | 群组 ID |
| `event_seq` | INTEGER | 群内单调序号 |
| `event_type` | TEXT | **§4.9 群事件类型**（`group_types.go`） |
| `actor_peer_id` | TEXT | 行为者 |
| `signer_peer_id` | TEXT | 签名者（可与 actor 不同） |
| `payload_json` | TEXT | 事件载荷 JSON |
| `signature` | BLOB | 签章 |
| `created_at` | TEXT | 建立时间 |

索引：`idx_group_events_group_seq`。

---

### 3.4 `group_epochs`

每世代群密钥包（给本机解密用）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | TEXT PK 复合 | 群组 ID |
| `epoch` | INTEGER PK 复合 | 世代号 |
| `wrapped_key_for_local` | BLOB | 本机可解开的 wrapped key |
| `created_at` | TEXT | 建立时间 |

---

### 3.5 `group_messages`

| 字段 | 类型 | 说明 |
|------|------|------|
| `msg_id` | TEXT PK | 消息 ID |
| `group_id` | TEXT | 群组 ID |
| `epoch` | INTEGER | 加密所用世代 |
| `sender_peer_id` | TEXT | 发送者 |
| `sender_seq` | INTEGER | 发送者于群内序号（与 `group_id`、`sender_peer_id` 唯一） |
| `msg_type` | TEXT | 群消息协议类型（如 `group_chat_text`、`group_chat_file` 等） |
| `plaintext` | TEXT | 解密后内容 |
| `file_name` / `mime_type` / `file_size` | TEXT/INTEGER | 文件消息字段 |
| `ciphertext_blob` | BLOB | 密文 |
| `signature` | BLOB | 消息签章 |
| `state` | TEXT | **§4.10 群消息状态** |
| `sent_at` | TEXT | 发送时间 |
| `delivered_summary` | TEXT | 送达汇总（JSON 字符串，默认 `''`） |

索引：`idx_group_messages_sender_seq`、`idx_group_messages_group_time`。

---

### 3.6 `group_message_deliveries`

每则群消息对每位成员的传递状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| `msg_id` | TEXT PK 复合 | 消息 ID |
| `peer_id` | TEXT PK 复合 | 成员 peer |
| `transport_mode` | TEXT | 传输模式 |
| `state` | TEXT | **§4.11 群传递状态** |
| `retry_count` | INTEGER | 重试次数 |
| `next_retry_at` | TEXT | 下次重试 |
| `delivered_at` | TEXT | 送达时间 |
| `updated_at` | TEXT | 更新时间 |

索引：`idx_group_deliveries_retry`。

---

### 3.7 `group_sync_cursors`

与各 peer 的群消息同步游标。

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | TEXT PK 复合 | 群组 ID |
| `peer_id` | TEXT PK 复合 | 对端 peer |
| `max_sender_seq` | INTEGER | 已同步之最大 `sender_seq` |
| `updated_at` | TEXT | 更新时间 |

---

### 3.8 `group_event_deliveries`

群事件对各 peer 的传递／重试状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | TEXT PK 复合 | 事件 ID |
| `peer_id` | TEXT PK 复合 | 目标 peer |
| `state` | TEXT | 传递状态（语义类似群消息递送） |
| `retry_count` | INTEGER | 重试次数 |
| `next_retry_at` | TEXT | 下次重试 |
| `updated_at` | TEXT | 更新时间 |

索引：`idx_group_event_deliveries_retry`。

---

### 3.9 `group_message_revocations`

已撤回的群消息记录（防再次展示等）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | TEXT PK 复合 | 群组 ID |
| `msg_id` | TEXT PK 复合 | 消息 ID |
| `sender_peer_id` | TEXT | 原发送者 |
| `revoked_by_peer_id` | TEXT | 撤回操作者 |
| `created_at` | TEXT | 撤回时间 |

## 4. 枚举与常见取值

### 4.1 好友请求状态 `requests.state`

| 值 | 常量 |
|----|------|
| `pending` | `RequestStatePending` |
| `accepted` | `RequestStateAccepted` |
| `rejected` | `RequestStateRejected` |
| `blocked` | `RequestStateBlocked` |

---

### 4.2 会话状态 `conversations.state`

| 值 | 说明 |
|----|------|
| `active` | 正常 |
| `no_friend` | 已非好友关系（例如收到拒绝后） |

---

### 4.3 传输模式 `*_transport_mode`

目前单聊常量：`direct`（`TransportModeDirect`）。

---

### 4.4 单聊 `messages.msg_type`（节选）

完整定义见 `types.go`：`session_request`、`session_accept`、`chat_text`、`chat_file`、`delivery_ack`、`message_revoke`、`retention_update` 等。

---

### 4.5 单聊消息状态 `messages.state` / 部分 job `status`

| 值 | 常量 |
|----|------|
| `local_only` | `MessageStateLocalOnly` |
| `sent_to_transport` | `MessageStateSentToTransport` |
| `delivered_remote` | `MessageStateDeliveredRemote` |
| `received` | `MessageStateReceived` |
| `queued_for_retry` | `MessageStateQueuedForRetry` |

---

### 4.6 群状态 `groups.state`

`active`、`archived`（`GroupStateActive` / `GroupStateArchived`）。

---

### 4.7 群角色 `group_members.role`

`controller`、`admin`、`member`。

---

### 4.8 成员状态 `group_members.state`

`invited`、`active`、`left`、`removed`。

---

### 4.9 群事件类型 `group_events.event_type`（节选）

`group_create`、`group_invite`、`group_join`、`group_leave`、`group_remove`、`group_title_update`、`group_retention_update`、`group_message_revoke`、`group_dissolve`、`group_controller_transfer`、`group_epoch_rotate` 等（`group_types.go`）。

---

### 4.10 群消息状态 `group_messages.state`

`local_only`、`sent_to_transport`、`queued_for_retry`、`partially_sent`、`delivered_remote`、`partially_delivered`、`failed_partial`、`received` 等（`GroupMessageState*`）。

---

### 4.11 群传递状态 `group_message_deliveries.state`

`pending`、`sent_to_transport`、`queued_for_retry`、`delivered_remote`、`failed`（`GroupDeliveryState*`）。

---

### 4.12 保留同步 `conversations.retention_sync_state`

程序中至少使用：`synced`（已与对端对齐）、`pending`（等待对端确认）。

---

## 5. 保留政策范围

程序常量：`MinRetentionMinutes`、`MaxRetentionMinutes`（`store.go`），用于约束 `retention_minutes` 合理区间；`0` 常表示「未设置／默认」语义，实际 UI 与协议行为以业务逻辑为准。
