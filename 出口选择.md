
用户可以配置每条电路最后一跳的 **Exit 节点选择条件**，例如：

* 只允许某个国家/地区的出口节点
* 优先某个国家/地区
* 指定某个具体 Exit 节点
* 没找到符合条件的 Exit 时如何降级

这个功能应该放在 **客户端本地路由策略 / Path Selector** 这一层，而不是放在 Exit 节点自身配置里。

因为这是“**我这边选谁做最后一跳**”，不是“**Exit 节点自己允许什么**”。

---

# 出口选择策略功能设计

## 1. 功能目标

允许本地节点在构建 circuit 时，控制最后一跳 Exit 的选择范围。

例如：

* 只选 `SG` 出口
* 优先 `TH`，找不到再用 `MY`
* 指定 `peer_id = 12D3Koo...` 做出口
* 匿名模式下必须用非本国出口
* 视频流量走日本出口，普通流量走自动选择

---

# 2. 放在哪一层

建议放在这两层：

## 2.1 配置层

配置文件定义用户的出口选择规则。

## 2.2 Path Selector 层

真正执行选择逻辑：

1. 收集所有可用 exit descriptor
2. 按规则过滤
3. 排序
4. 选出最终最后一跳 exit

---

# 3. 你真正需要的配置能力

我建议至少支持 4 类模式。

## 模式 A：自动

系统自动选 exit。

```yaml
exit_selection:
  mode: auto
```

---

## 模式 B：只允许某个国家/地区

例如：

```yaml
exit_selection:
  mode: country_only
  allowed_countries: ["SG"]
```

含义：

* 最后一跳 exit 只能从 `SG` 节点里选
* 其他国家一律不选

这就是你说的：

**“仅选择来自某个特定国家/地区的出口中继”**

---

## 模式 C：国家优先

例如：

```yaml
exit_selection:
  mode: country_preferred
  preferred_countries: ["TH", "SG", "MY"]
  fallback_to_any: true
```

含义：

* 优先 TH
* 没有再试 SG
* 再没有试 MY
* 最后如果还没有，允许退回任意国家

---

## 模式 D：指定具体 Exit 节点

例如：

```yaml
exit_selection:
  mode: fixed_peer
  fixed_exit_peer_id: "12D3KooWxxxx"
```

含义：

* 每条电路最后一跳必须用这个 exit
* 找不到就失败或按 fallback 规则处理

---

# 4. 推荐最终配置结构

建议你把它做成一组统一配置。

```yaml
client:
  exit_selection:
    mode: country_only   # auto | country_only | country_preferred | fixed_peer

    allowed_countries: ["SG"]
    preferred_countries: ["TH", "SG", "MY"]

    fixed_exit_peer_id: ""

    exclude_countries: ["CN"]
    exclude_peer_ids: []

    require_remote_dns: true
    require_tcp_support: true

    fallback_to_any: false
    allow_direct_exit: true
```

---

# 5. 字段含义设计

## `mode`

支持：

* `auto`
* `country_only`
* `country_preferred`
* `fixed_peer`

---

## `allowed_countries`

只在 `country_only` 模式下强约束。

例如：

```yaml
allowed_countries: ["SG", "JP"]
```

表示最后一跳只能从这些国家里选。

---

## `preferred_countries`

只在 `country_preferred` 模式下使用。

例如：

```yaml
preferred_countries: ["TH", "SG"]
```

表示先按 TH、再 SG 的顺序挑选。

---

## `fixed_exit_peer_id`

固定最后一跳的 Exit Peer ID。

---

## `exclude_countries`

无论哪种模式，都排除这些国家的出口。

---

## `exclude_peer_ids`

屏蔽某些特定 exit 节点。

---

## `require_remote_dns`

如果为 `true`，则只选支持 remote DNS 的 exit。

---

## `require_tcp_support`

如果为 `true`，则只选支持 TCP 的 exit。

---

## `fallback_to_any`

当目标国家没有可用出口时，是否降级到“任意出口”。

* `true`：允许退化
* `false`：直接失败

---

## `allow_direct_exit`

是否允许路径为：

`Local -> Exit`

如果为 `false`，则即使选定 exit，也必须至少经过一个 relay。

---

# 6. Path Selector 逻辑设计

这是核心。

## 6.1 选择流程

假设要为一条 circuit 选最后一跳 exit，建议顺序：

### Step 1：获取所有可用 exits

从 discovery store 读取所有 `roles contains exit` 的节点。

### Step 2：基础能力过滤

先过滤掉不满足最低要求的：

* 不支持 TCP
* 不支持 remote DNS（如果 require）
* 本地排除的 peer_id
* 本地排除的 country
* 自己节点（默认不选自己）

### Step 3：按 mode 再过滤

#### `auto`

不额外限制

#### `country_only`

只保留 `country in allowed_countries`

#### `country_preferred`

按 `preferred_countries` 分组排序

#### `fixed_peer`

只保留 `peer_id == fixed_exit_peer_id`

### Step 4：评分排序

对过滤后的 exit 按评分排序：

* 成功率
* RTT
* 带宽
* 最近失败
* 当前负载（如果有）

### Step 5：选择最终 exit

返回最高分候选。

# 7. 失败与降级策略

这个一定要设计清楚。

## 7.1 严格模式

例如：

```yaml
mode: country_only
allowed_countries: ["SG"]
fallback_to_any: false
```

如果没有 SG exit，就直接失败。

---

## 7.2 软降级模式

例如：

```yaml
mode: country_preferred
preferred_countries: ["TH", "SG"]
fallback_to_any: true
```

先找 TH，没有找 SG，再没有就任意国家。

---

## 7.3 固定节点失效

例如：

```yaml
mode: fixed_peer
fixed_exit_peer_id: "12D3Koo..."
fallback_to_any: false
```

如果该节点下线，直接失败。

或者：

```yaml
fallback_to_any: true
```

则退回 सामान्य自动选择。

---

# 8. Descriptor 需要包含的字段

为了支持这个功能，Exit 节点 descriptor 至少要有：

```json
{
  "roles": ["relay", "exit"],
  "exit": {
    "allow_tcp": true,
    "allow_udp": false,
    "allowed_ports": [80, 443],
    "country": "SG",
    "city": "Singapore",
    "remote_dns": true,
    "bandwidth_mbps": 100,
    "tags": ["stable", "tcp"]
  }
}
```

其中对 exit 选择最关键的是：

* `country`
* `remote_dns`
* `allow_tcp`
* `tags`
* `peer_id`

# 9. 路由规则里也要支持出口国家

这个功能最好不只是“全局设置”，还应该能放进 Routing Rules。

例如：

```yaml
routing:
  rules:
    - name: video-jp
      domain_suffix: ".netflix.com"
      mode: low_latency
      exit_country: JP
      priority: 100

    - name: banking-sg
      domain_suffix: ".bank.example"
      mode: anonymous
      exit_country: SG
      priority: 90

    - name: default
      mode: anonymous
      exit_country: TH
      priority: 1
```

这样不同流量可以走不同国家的出口。

# 10. 建议的数据结构

## `ExitSelectionConfig`

```go
type ExitSelectionMode string

const (
    ExitSelectionAuto             ExitSelectionMode = "auto"
    ExitSelectionCountryOnly      ExitSelectionMode = "country_only"
    ExitSelectionCountryPreferred ExitSelectionMode = "country_preferred"
    ExitSelectionFixedPeer        ExitSelectionMode = "fixed_peer"
)

type ExitSelectionConfig struct {
    Mode               ExitSelectionMode `yaml:"mode" json:"mode"`
    AllowedCountries   []string          `yaml:"allowed_countries" json:"allowed_countries"`
    PreferredCountries []string          `yaml:"preferred_countries" json:"preferred_countries"`
    FixedExitPeerID    string            `yaml:"fixed_exit_peer_id" json:"fixed_exit_peer_id"`

    ExcludeCountries   []string          `yaml:"exclude_countries" json:"exclude_countries"`
    ExcludePeerIDs     []string          `yaml:"exclude_peer_ids" json:"exclude_peer_ids"`

    RequireRemoteDNS   bool              `yaml:"require_remote_dns" json:"require_remote_dns"`
    RequireTCPSupport  bool              `yaml:"require_tcp_support" json:"require_tcp_support"`

    FallbackToAny      bool              `yaml:"fallback_to_any" json:"fallback_to_any"`
    AllowDirectExit    bool              `yaml:"allow_direct_exit" json:"allow_direct_exit"`
}
```

建议补两个接口。

`GET /api/v1/client/exit-selection`

---

## 更新出口选择配置

`POST /api/v1/client/exit-selection`

例如传：

```json
{
  "mode": "country_only",
  "allowed_countries": ["SG"],
  "fallback_to_any": false
}
```

---

## 查看候选出口

`GET /api/v1/client/exit-candidates`

返回：

* 当前规则下可选 exits
* 被过滤原因
* 最终选中哪个

这个接口非常有用，调试时能立刻知道为什么没选到某个出口。

这样排问题会很容易。
